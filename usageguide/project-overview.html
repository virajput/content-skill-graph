What it does

  • Starts every workflow from a command-center index.
  • Defines platform-native writing rules per channel.
  • Separates brand voice from platform tone adaptation.
  • Gives you hook formulas, batch workflow, and schedule logic.
  • Works as a modular graph for Claude, Claude Code, or Obsidian.

Quick start with Claude Code

cd content-skill-graph-project

Prompt:
Read content-skill-graph/index.md first.
Follow the linked markdown files relevant to the topic.
Generate 8 platform-native content assets.
Save them under outputs/[topic-slug]/ with one file per platform.

Best use case

Use this when you want one topic to become multiple platform-specific assets without sounding duplicated. The graph forces context, tone control, and proper repurposing.
